## 👥 USER ROLES — EXACTLY 2

### ROLE 1: OWNER (Admin)
Full access to everything. Login with username + password.

### ROLE 2: CASHIER
Restricted access. Can only:
- Login
- Create bills (manual entry + barcode)
- Search products
- Add new product during billing (triggers owner notification)
- View their own sales history
- View current stock levels (read only)

Cashier CANNOT:
- View investment/profit reports
- Delete products
- Edit prices
- Restore backups
- View other cashiers' data

## 🗄️ DATABASE SCHEMA — PostgreSQL

Create ALL tables with these exact columns. Use `UUID` as primary keys.
Run via a `schema.sql` file with `IF NOT EXISTS`.

```sql
-- USERS
CREATE TABLE users (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  name VARCHAR(100) NOT NULL,
  username VARCHAR(50) UNIQUE NOT NULL,
  password_hash VARCHAR(255) NOT NULL,
  role VARCHAR(10) NOT NULL CHECK (role IN ('owner','cashier')),
  phone VARCHAR(15),
  is_active BOOLEAN DEFAULT true,
  created_at TIMESTAMPTZ DEFAULT NOW(),
  updated_at TIMESTAMPTZ DEFAULT NOW()
);

-- CATEGORIES
CREATE TABLE categories (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  name_en VARCHAR(100) NOT NULL,
  name_ta VARCHAR(100),
  created_at TIMESTAMPTZ DEFAULT NOW()
);

-- PRODUCTS
CREATE TABLE products (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  name_en VARCHAR(200) NOT NULL,
  name_ta VARCHAR(200),
  category_id UUID REFERENCES categories(id),
  barcode VARCHAR(100) UNIQUE,
  unit_type VARCHAR(20) NOT NULL CHECK (unit_type IN ('qty','weight_kg','weight_g','litre','ml','packet','box','dozen')),
  purchase_price NUMERIC(10,2) NOT NULL DEFAULT 0,
  selling_price NUMERIC(10,2) NOT NULL DEFAULT 0,
  current_stock NUMERIC(10,3) NOT NULL DEFAULT 0,
  min_stock_alert NUMERIC(10,3) DEFAULT 5,
  hsn_code VARCHAR(20),
  gst_rate NUMERIC(5,2) DEFAULT 0,
  is_active BOOLEAN DEFAULT true,
  created_by UUID REFERENCES users(id),
  created_at TIMESTAMPTZ DEFAULT NOW(),
  updated_at TIMESTAMPTZ DEFAULT NOW()
);

-- STOCK MOVEMENTS (every stock change is logged here)
CREATE TABLE stock_movements (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  product_id UUID REFERENCES products(id),
  movement_type VARCHAR(20) NOT NULL CHECK (movement_type IN ('purchase','sale','return','adjustment','damaged','expired','opening')),
  quantity NUMERIC(10,3) NOT NULL,
  unit_price NUMERIC(10,2),
  total_value NUMERIC(12,2),
  reference_id UUID,          -- bill_id or purchase_id
  notes TEXT,
  performed_by UUID REFERENCES users(id),
  movement_date TIMESTAMPTZ DEFAULT NOW()
);

-- PURCHASES (Stock coming IN — investment tracking)
CREATE TABLE purchases (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  supplier_name VARCHAR(200),
  supplier_phone VARCHAR(15),
  invoice_number VARCHAR(100),
  purchase_date DATE NOT NULL DEFAULT CURRENT_DATE,
  total_amount NUMERIC(12,2) NOT NULL DEFAULT 0,
  paid_amount NUMERIC(12,2) DEFAULT 0,
  payment_mode VARCHAR(20) DEFAULT 'cash' CHECK (payment_mode IN ('cash','upi','credit','bank_transfer')),
  notes TEXT,
  created_by UUID REFERENCES users(id),
  created_at TIMESTAMPTZ DEFAULT NOW()
);

-- PURCHASE ITEMS
CREATE TABLE purchase_items (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  purchase_id UUID REFERENCES purchases(id) ON DELETE CASCADE,
  product_id UUID REFERENCES products(id),
  product_name VARCHAR(200) NOT NULL,
  quantity NUMERIC(10,3) NOT NULL,
  unit_price NUMERIC(10,2) NOT NULL,
  total_price NUMERIC(12,2) NOT NULL
);

-- BILLS (Sales)
CREATE TABLE bills (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  bill_number VARCHAR(30) UNIQUE NOT NULL,   -- SMS-YYYYMMDD-0001
  customer_name VARCHAR(100),
  customer_phone VARCHAR(15),
  subtotal NUMERIC(12,2) NOT NULL DEFAULT 0,
  discount_amount NUMERIC(10,2) DEFAULT 0,
  gst_amount NUMERIC(10,2) DEFAULT 0,
  total_amount NUMERIC(12,2) NOT NULL DEFAULT 0,
  paid_amount NUMERIC(12,2) DEFAULT 0,
  change_amount NUMERIC(10,2) DEFAULT 0,
  payment_mode VARCHAR(20) DEFAULT 'cash' CHECK (payment_mode IN ('cash','upi','card','credit')),
  bill_status VARCHAR(20) DEFAULT 'paid' CHECK (bill_status IN ('paid','credit','cancelled')),
  cashier_id UUID REFERENCES users(id),
  bill_date TIMESTAMPTZ DEFAULT NOW(),
  notes TEXT
);

-- BILL ITEMS
CREATE TABLE bill_items (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  bill_id UUID REFERENCES bills(id) ON DELETE CASCADE,
  product_id UUID REFERENCES products(id),
  product_name VARCHAR(200) NOT NULL,
  unit_type VARCHAR(20) NOT NULL,
  quantity NUMERIC(10,3) NOT NULL,
  unit_price NUMERIC(10,2) NOT NULL,
  purchase_price NUMERIC(10,2) NOT NULL,  -- snapshot for profit calc
  discount NUMERIC(10,2) DEFAULT 0,
  gst_rate NUMERIC(5,2) DEFAULT 0,
  gst_amount NUMERIC(10,2) DEFAULT 0,
  total_price NUMERIC(12,2) NOT NULL
);

-- EXPENSES
CREATE TABLE expenses (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  category VARCHAR(100) NOT NULL,  -- rent, electricity, salary, transport, misc
  description TEXT,
  amount NUMERIC(10,2) NOT NULL,
  expense_date DATE NOT NULL DEFAULT CURRENT_DATE,
  payment_mode VARCHAR(20) DEFAULT 'cash',
  created_by UUID REFERENCES users(id),
  created_at TIMESTAMPTZ DEFAULT NOW()
);

-- AUDIT LOGS
CREATE TABLE audit_logs (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  user_id UUID REFERENCES users(id),
  action VARCHAR(100) NOT NULL,
  table_name VARCHAR(50),
  record_id UUID,
  old_value JSONB,
  new_value JSONB,
  ip_address VARCHAR(50),
  created_at TIMESTAMPTZ DEFAULT NOW()
);

-- NOTIFICATIONS
CREATE TABLE notifications (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  for_role VARCHAR(10) DEFAULT 'owner',
  type VARCHAR(50),    -- new_product, low_stock, credit_bill
  message TEXT NOT NULL,
  reference_id UUID,
  is_read BOOLEAN DEFAULT false,
  created_at TIMESTAMPTZ DEFAULT NOW()
);

-- SETTINGS
CREATE TABLE settings (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  key VARCHAR(100) UNIQUE NOT NULL,
  value TEXT,
  updated_at TIMESTAMPTZ DEFAULT NOW()
);

-- Seed default settings
INSERT INTO settings (key, value) VALUES
  ('store_name', 'Sri Murugan Store'),
  ('store_address', ''),
  ('store_phone', ''),
  ('store_gstin', ''),
  ('store_state', 'Tamil Nadu'),
  ('currency', '₹'),
  ('gst_enabled', 'true'),
  ('low_stock_alert_enabled', 'true'),
  ('receipt_footer', 'Thank you! வாருங்கள் மீண்டும்!'),
  ('bill_prefix', 'SMS')
ON CONFLICT (key) DO NOTHING;
```

## 🔐 AUTHENTICATION — FULL IMPLEMENTATION

### POST /api/v1/auth/login
```json
Request:  { "username": "owner", "password": "password123" }
Response: {
  "user": { "id", "name", "username", "role" },
  "accessToken": "...",   // expires 15min
  "refreshToken": "..."   // expires 7 days, store in httpOnly cookie
}
```

### POST /api/v1/auth/refresh
- Read refreshToken from httpOnly cookie
- Return new accessToken

### POST /api/v1/auth/logout
- Clear cookie

### Frontend auth flow:
- Store accessToken in Zustand (memory only, not localStorage)
- On 401, auto-call refresh endpoint, retry original request
- Redirect to /login if refresh fails

### First-launch setup:
- If zero users in DB → show "Create Owner Account" screen before login
- Fields: Store Name, Owner Name, Username, Password, Store Phone, GSTIN (optional)

When search returns 0 results, show button:
**"+ Add New Product Manually"**

Opens a modal with fields:
- Product Name (English) — required
- Product Name (Tamil) — optional
- Category — dropdown, required
- Unit Type — dropdown: qty / weight(kg) / weight(g) / litre / ml / packet / box

- Purchase Price (Cost) — required
- Selling Price — required
- Stock to add — required

On Save:
1. Create product in DB immediately
2. Add stock_movement (type: opening)
3. Add to current bill automatically
4. Send notification to owner: "Cashier [name] added new product: [name]"
5. Log in audit_logs

#### Cart behavior:
- Each item shows: Name, Unit, Qty (editable inline), Price, Line Total, [Delete]
- Qty can be decimal for weight/litre (e.g. 1.5 kg)
- Changing qty recalculates line total instantly
- Remove item with × button
- Cart persists if user navigates away (Zustand)

- Name (text)

## 📦 STOCK MANAGEMENT MODULE

### Stock Tracking — Every movement logged:

| Movement Type | Trigger                     |
|---------------|-----------------------------|
| `purchase`    | When purchase entry saved   |
| `sale`        | When bill is created        |
| `return`      | Manual stock return entry   |
| `adjustment`  | Manual correction by owner  |
| `damaged`     | Mark items as damaged       |
| `expired`     | Mark expired stock          |
| `opening`     | When new product added      |

### API Endpoints:
```
GET  /api/v1/stock/current          - all products with current stock
GET  /api/v1/stock/low-alert        - products below min_stock_alert
GET  /api/v1/stock/movements        - paginated, filterable by product/date
POST /api/v1/stock/adjust           - owner only
POST /api/v1/stock/mark-damaged     - owner only
GET  /api/v1/stock/valuation        - total inventory value (stock × purchase_price)
```

### Stock Dashboard widget (Owner):

## 🛒 PURCHASES MODULE (Stock Coming IN)

### Purpose: Track investment — how much money spent on buying stock

### Form fields:
- Supplier Name
- Supplier Phone
- Invoice/Bill Number (supplier's bill no.)
- Purchase Date (default today)
- Payment Mode: Cash / UPI / Credit / Bank Transfer
- Items table:
  - Search & select product OR type new product name
  - Quantity
  - Unit Price (purchase cost)
  - Total (auto)
- Grand Total (auto)
- Notes

### On saving purchase:
1. Update `products.current_stock` += quantity for each item
2. Update `products.purchase_price` = latest unit price
3. Insert `stock_movements` (type: purchase)
4. Insert `purchase_items` records

### API:
```
GET  /api/v1/purchases              - list, filter by date/supplier
POST /api/v1/purchases              - create
GET  /api/v1/purchases/:id          - single with items
GET  /api/v1/purchases/summary      - monthly totals
```

- CGST = half of GST rate (Central)
- SGST = half of GST rate (State — Tamil Nadu)
- Standard GST slabs: 0%, 5%, 12%, 18%, 28%
- GSTIN format validation: `33XXXXX1234X1ZX` (33 = Tamil Nadu state code)
- HSN code field for each product

### Date & Time:
- Display: `DD/MM/YYYY` (Indian format)

- All dates stored as UTC in DB, displayed in IST (UTC+5:30)

## ⚠️ CRITICAL RULES FOR THE CODING AI

1. **Build every feature listed. Do not skip anything.**

2. **Billing must work without barcode** — manual search is primary.

3. **All monetary calculations must use server-side precision** (NUMERIC in DB, not float in JS).

4. **Stock is always deducted at bill save time**, never at cart time.

5. **bill_items must snapshot purchase_price at time of billing** so profit can be calculated correctly even if prices change later.

6. **All amounts in Indian Rupees (₹)** — use `en-IN` locale for formatting.

7. **Dates stored as UTC, displayed as IST (Asia/Kolkata, UTC+5:30)**.

8. **GST split: CGST + SGST** (Tamil Nadu intra-state). No IGST for this store.

9. **Every stock change must write to stock_movements table** — this is the audit trail.

10. **Cashier can add new product during billing** — this is a non-negotiable feature.

11. **Receipt must print correctly on 80mm thermal printer.**

12. **Zero TypeScript `any` types** — define proper types in `/types/`.

13. **Error handling everywhere** — all API errors return `{ success: false, message: string }`.

14. **All sensitive routes must check JWT + role** before executing.

15. **The app must be fully runnable with `docker compose up`.**
