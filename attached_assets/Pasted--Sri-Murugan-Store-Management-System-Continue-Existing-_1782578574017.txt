### 🟠 P1 — Manual Cash Drawer & Daily Cashout (End-of-Day Settlement)

#### Overview
Add an end-of-day "Cash Drawer" feature that lets the owner/cashier:
1. **View today's total sales** broken down by payment mode (cash, UPI, card, credit) — pulled live from DB
2. **Manually record each cash denomination** counted in the physical drawer (optional, can skip)
3. **Record a cashout** (close the day) with actual cash in hand vs expected — stored for audit
4. **View cashout history** per day / per cashier

---

#### DB Migration (use `IF NOT EXISTS` — never DROP)
```sql
-- Daily cashout records
CREATE TABLE IF NOT EXISTS cashouts (
  id              SERIAL PRIMARY KEY,
  cashier_id      INTEGER NOT NULL REFERENCES users(id),
  cashout_date    DATE NOT NULL DEFAULT CURRENT_DATE,
  
  -- Expected (from bills)
  expected_cash   NUMERIC(10,2) NOT NULL DEFAULT 0,
  expected_upi    NUMERIC(10,2) NOT NULL DEFAULT 0,
  expected_card   NUMERIC(10,2) NOT NULL DEFAULT 0,
  expected_credit NUMERIC(10,2) NOT NULL DEFAULT 0,
  
  -- Actual (manually entered)
  actual_cash     NUMERIC(10,2),   -- NULL = denomination counting skipped
  notes           TEXT,
  cash_difference NUMERIC(10,2) GENERATED ALWAYS AS (actual_cash - expected_cash) STORED,
  
  -- Denomination breakdown (optional JSON)
  denomination_breakdown JSONB,  
  -- e.g. {"2000":1,"500":3,"200":2,"100":5,"50":4,"20":3,"10":10,"5":5,"2":2,"1":10}
  
  status          VARCHAR(20) DEFAULT 'open' CHECK (status IN ('open','closed')),
  closed_at       TIMESTAMP,
  created_at      TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

-- Only one open cashout per cashier per day
CREATE UNIQUE INDEX IF NOT EXISTS idx_cashouts_cashier_date 
  ON cashouts(cashier_id, cashout_date) WHERE status = 'open';
```

**Step 2 — Denomination Counter (optional, collapsible)**
```
[ Count Cash Denominations ]  ▼ (toggle open/close)

  ₹ 2000  × [  1  ]  =  ₹  2000
  ₹  500  × [  3  ]  =  ₹  1500
  ₹  200  × [  2  ]  =  ₹   400
  ₹  100  × [  5  ]  =  ₹   500
  ₹   50  × [  4  ]  =  ₹   200
  ₹   20  × [  3  ]  =  ₹    60
  ₹   10  × [ 10  ]  =  ₹   100
  ₹    5  × [  5  ]  =  ₹    25
  ₹    2  × [  2  ]  =  ₹     4
  ₹    1  × [ 10  ]  =  ₹    10
  ─────────────────────────────
  Counted Cash Total:   ₹ 4,799
```
- Each denomination row: label + controlled number input (min 0, keyboard-friendly)
- Total auto-calculates as user types (no submit needed for this step)
- Denomination notes: coins (₹1, ₹2, ₹5) counted separately

**Step 3 — Actual Cash Entry + Notes**
```
  Actual Cash in Drawer:  [ ₹ _______ ]   ← auto-filled if denominations used, or manual
  Notes (optional):       [ ___________________________ ]

  Expected Cash:  ₹ 4,250.00
  Actual Cash:    ₹ 4,799.00
  Difference:     ₹  +549.00  ✅  (or ❌ if negative)

              [ Submit Cashout ]
```
- If denomination section was used, `actual_cash` auto-fills with denomination total
- If denomination skipped, manual entry of actual cash
- Difference shown live (color-coded: green = surplus, red = shortage, grey = exact)
- Submit → `POST /cashouts` → success toast → page shows "Cashout closed" state

#### Error & Edge Case Handling
| Scenario | Behavior |
|---|---|
| No bills today | Summary shows ₹0 across all modes; cashout still recordable |
| Cashout already submitted today | Button disabled, shows timestamp; owner can still view |
| `actual_cash` left blank | Saved as NULL (means "not counted") — valid submission |
| Denomination inputs accept only integers ≥ 0 | Validate on frontend + backend |
| Negative stock from cancelled bills after cashout | Cashout is immutable once closed; owner sees note in history |
